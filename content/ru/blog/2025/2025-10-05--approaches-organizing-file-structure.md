Подходы к организации структуры файлов.

<!--more-->

{{< toc >}}


## <span class="section-num">1</span> Иерархическая организация {#иерархическая-организация}

-   Папки по проектам
    -   Создание отдельной папки для каждого проекта
    -   Вложенность по этапам работы
    -   Пример: `Проект_X/Дизайн/Концепты`

-   Категорийная структура
    -   Группировка по типам файлов (документы, изображения, видео)
    -   Подкатегории по назначению
    -   Пример: `Документы/Работа/2024`

-   Временная организация
    -   Сортировка по датам создания
    -   Годовая/месячная структура
    -   Пример: `2024/09_Сентябрь/Файлы`


## <span class="section-num">2</span> Системы именования {#системы-именования}

-   Стандартные соглашения
    -   Формат: дата_проект_версия
    -   Использование подчеркиваний вместо пробелов
    -   Пример: `2024-09-03_Отчет_v2.pdf`

-   Метаданные в названиях
    -   Включение ключевых характеристик
    -   Использование разделителей
    -   Пример: `Проект-X_Финальный_Версия2_2024`


## <span class="section-num">3</span> Теггирование и метаданные {#теггирование-и-метаданные}

-   Система меток
    -   Добавление тегов к файлам
    -   Поиск по ключевым словам
    -   Пример: `#проект #важное #2024`

-   Расширенные метаданные
    -   Описание содержимого
    -   Ключевые слова
    -   Автор и дата создания


## <span class="section-num">4</span> Специализированные системы {#специализированные-системы}

-   Johnny Decimal
    -   [Система именования каталогов Johnny Decimal]({{< relref "2025-07-11--johnny-decimal-naming-system" >}})

    -   Числовая иерархия
    -   Строгая структура
    -   Пример: `10.01_Личные_Документы`

-   Pera (PARA)
    -   Projects (проекты)
    -   Areas (области)
    -   Resources (ресурсы)
    -   Archive (архив)

-   Zettelkasten
    -   Атомарные заметки
    -   Связывание идей
    -   Уникальные номера


## <span class="section-num">5</span> Гибридные методы {#гибридные-методы}

-   Комбинированная структура
    -   Иерархия + теги
    -   Числовая система + категории
    -   Пример: `10.01_Документы_2024 #работа`

-   Адаптивные системы
    -   Гибкая настройка под задачи
    -   Модульный подход
    -   Возможность масштабирования
